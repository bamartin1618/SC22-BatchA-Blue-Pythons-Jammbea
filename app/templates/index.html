Step 1

In the end, we used two datasets to train our AI. However, at first, our team searched for a dataset with quality python code. We knew that better data for training the AI would result in a better product. Finally, we found a dataset on Kaggle, containing more than enough code to train our model-or so we thought. The dataset, which was called the 150k Python Dataset from SRI-LAB, contained about 88 MB of Python code, but we took a smaller sample). In our evaluation, we found that quality was probably more important than quantity, as our dataset had contained too much OOP (Object-Oriented-Programming) which our model had a hard time understanding.
After our evaluation, we found the second dataset, which was called the english_python_data.txt. It contained only 1.1 MB of code, but had standard, simple Python code.

Step 2

The code in the first dataset was stored in a series of nested folders. Using recursion in Python, we retrieved the data from the folders and concatenated it into a text file. We also tried doing this in the Linux Terminal. Next, we had to split the data into smaller parts because it was too large of a dataset. We used random in Python to get random samples of the data.
Lastly, we found that our data had to be cleaned. Our team members worked together and removed the docstrings and comments from the code, using the Python re (Regular Expression) module. The second dataset was cleaned the same way, but there was no need to concatenate separate files since it was already a .txt file. Then, we combined the two datasets to create a single, useable text file that could be used to train our model.

Step 3

The next step was to train the AI model. Since we used Aitextgen, a library in Python that is used to generate text, we only needed to fine tune our model using the training data that we gathered before. We chose the gpt-neo model because it is known to get generally good results. Also, we were restrained by the amount of RAM google Colab had, so we could only use a smaller model. We started off with the standard parameters and after seeing the results, we trained the model the second time with a better-processed dataset and different parameters that gave more useable results. We set the model to train for 12,000 steps. Our learning rate was set to 5e-3, and the AI generated text every 1000 steps.

Step 4

We tested out our AI Code Generator manually by entering different prompts in Google Colab. Based on the results, we processed the data again, and added more data. We found the better quality dataset (previously mentioned as the "second dataset"), though it had less in quantity, and cleaned it, adding it to the original dataset. Then, we trained the AI, and re-evaluated. We used autopep8 as a python linter, which helped format the code.

Step 5

We created the website using Bootstrap and revised it in html code. More features were added, such as Prism.js which highlighted the output according to Python syntax, a working copy button, and a dark-mode toggle. Collaborating on Replit, and then on Cocalc, we made the front-end of our product.

Step 6

Using Flask, the website and the AI parts of our product were merged together. This is where we are now. In the future, we plan to add a python compiler so the user is able to run the code in the website without requiring them to copy the code and find a compiler of their own. We also will allow the dark mode to be saved automatically to the user's system, so the user only needs to change modes once instead of every time the website is reloaded. A plan is to test out more auto-correctors. We want the code to run every time, so we need a better corrector to adjust the outputted code.
With additional time, a good idea would be to clean the dataset more, by removing all of the code that had the potential to cause errors, so the model would train off of ONLY functioning code which would reduce the amount of mistakes and errors.
